## 1. 先搞懂问题本身

### 1.1 物理场景

想象你有15块大木头（3种型号，每种5块）。你要从这些木头上切割出小零件卖钱。

- 零件可以90度旋转（比如 40×40×40 的立方体，怎么转都一样；但 50×40×40 的长方体，有6种转法）
- 零件不能互相重叠（切出来的位置不能打架）
- 零件不能超出木头的边界
- 假设切缝没有损耗（理想情况）

### 1.2 两个问题

| | 问题1 | 问题2 |
|------|------|------|
| **目标** | 利用率最高 = 浪费最少 | 总利润最大 |
| **约束** | 每种木头正好用5块 | 每种木头正好用5块 **+ 每种零件至少切10个** |
| **难度** | 相对简单（只管填满） | 更难（要考虑利润和最低数量） |

### 1.3 核心数据直觉

最重要的概念是 **利润密度 = 利润 ÷ 体积**。它告诉你"同样的体积，切哪种零件最赚钱"。

```
J07 > J05 > J04 > J06 > J03 > J02 > J01（从高到低）
0.01125                                   0.00969
```

但利润密度高≠应该全切它，因为还要受限于：
- 几何能不能放进去
- 问题2每种至少10个的最低要求

### 1.4 问题规模

- 3种材料 × 5块 = 15块原材料
- 7种零件类型
- 一块材料最多能放几百个零件（取决于零件大小）
- 可能的切割方案数量：**天文数字**（NP-hard问题）

所以不能暴力枚举所有方案，必须用启发式+优化的组合方法。

## 2. 整体思路：为什么是"两步走"

### 2.1 核心洞察

如果直接对"每个零件的坐标"建数学模型，变量太多，求解器会爆炸。

所以我们把问题拆成两步：

```
┌─────────────────────────────────────────────────────────────┐
│                    第1步：方案生成                            │
│   输入：材料尺寸 + 零件尺寸                                    │
│   输出：几百种"切割方案"（每种方案 = 一块材料的完整切法）         │
│   方法：网格填充 + 贪心放置 + 极端点启发式                      │
└──────────────────────────┬──────────────────────────────────┘
                           │ 几百种候选方案
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                    第2步：方案选择                            │
│   输入：候选方案库                                             │
│   输出：从库中选哪些方案、各用几次                              │
│   方法：整数规划 → CP-SAT 求解器                               │
└──────────────────────────┬──────────────────────────────────┘
                           │ 最优方案组合
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                    第3步：结果输出                            │
│   图表生成 + Excel填写 + 结果验证 + 最优性证明                  │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 为什么要这样？类比理解

就像是**先准备菜单，再点菜**：

- 第1步 = 准备菜单：大厨（启发式算法）根据食材创造了100道菜
- 第2步 = 点菜：你（CP-SAT）从菜单里挑出最优的15道菜组合

你不是在"所有可能的菜"里挑（那太困难了），而是在"已经做好的100道菜"里挑。

**代价**：如果菜单里没有"完美的菜"，你就得不到全局最优。这就是为什么我们只能说"方案库内最优"而不是"全局最优"。

## 4. 数据是怎么流动的

### 4.1 一条请求的完整生命周期

```
1. 用户运行 python run_all.py
         │
2. run_all.py → run_all_experiments.main()
         │
3. 读取 configs/default.yaml（材料数据、算法参数）
         │
4. data.py: create_materials(), create_pieces()
   把YAML变成Python对象
   Material(name="L01", length=300, width=200, height=150, count=5)
   Piece(name="J01", length=40, width=40, height=40, profit=620)
         │
5. orientations.py: generate_all_orientations()
   为每种零件生成所有90度旋转姿态
   比如 J02(50×40×40) → 6种：50×40×40, 40×50×40, 40×40×50, ...
         │
6. pattern_generator.py: generate_all_patterns()
   对每种材料 × 每种方法，生成几百个候选切割方案
   每个方案 = 一堆PlacedPiece的列表（什么零件、放在哪里、什么姿态）
         │
7. master_solver.py: solve_master_problem1/2()
   构建CP-SAT模型：变量=方案使用次数，约束=材料数量，目标=利用率/利润
   求解 → 得到最优方案组合
         │
8. 验证：geometry.validate_pattern()
   逐一检查选中的方案：所有零件在内部？无重叠？
         │
9. 计算指标：bounds.py + metrics.py
   利用率、浪费、利润、gap、上下界
         │
10. 输出：visualization.py + excel_writer.py
    图表 → outputs/figures/
    Excel → outputs/results/result1_filled.xlsx
    JSON → outputs/results/problem1_solution.json
```

### 4.2 核心数据结构关系

```
Material（材料）
  ├── name, length, width, height, count
  └── volume（计算属性）

Piece（零件）
  ├── name, length, width, height, profit
  ├── volume（计算属性）
  └── profit_density（计算属性 = profit/volume）

Orientation（旋转姿态）
  └── dx, dy, dz（旋转后的三边长度）

PlacedPiece（已放置的零件 = 零件 + 位置 + 姿态）
  ├── piece_name
  ├── x, y, z（在材料中的坐标）
  └── orientation: Orientation

Pattern（切割方案 = 一堆PlacedPiece的集合）
  ├── pattern_id
  ├── material_name（这种方案适用于哪种材料）
  ├── placed_pieces: list[PlacedPiece]
  ├── used_volume（所有零件体积之和）
  └── total_profit（所有零件利润之和）

MasterSolution（最终解）
  ├── status: OPTIMAL / FEASIBLE / INFEASIBLE
  ├── pattern_usage: {pattern_id: 使用次数}
  ├── total_profit, total_waste_volume
  ├── material_utilization
  ├── piece_counts: {piece_name: 总生产数量}
  └── solve_time_seconds
```

## 5. 第1步详解：怎么生成切割方案

这一步由 `pattern_generator.py` 的 `generate_all_patterns()` 调度。

```python
# 对每种材料，分别用三种方法生成方案
for material in materials:
    网格方案 = generate_grid_patterns(material, ...)     # 方法A
    贪心方案 = generate_greedy_patterns(material, ...)   # 方法B
    极端点方案 = generate_ep_patterns(material, ...)     # 方法C
    all_patterns[material] = 合并去重(网格+贪心+极端点)
```

### 5.1 方法A：网格方案（Grid Patterns）

**思路**：对单一零件类型，按规则网格紧密排列。

**具体步骤**：
1. 对每种零件（J01~J07），对它的每种旋转姿态
2. 计算材料在各个方向上能放几个：
   - `nx = 材料长 // 零件长`（比如 300 // 60 = 5）
   - `ny = 材料宽 // 零件宽`（比如 200 // 50 = 4）
   - `nz = 材料高 // 零件高`（比如 150 // 30 = 5）
3. 最大容量 = nx × ny × nz（比如 5×4×5 = 100个）
4. 尝试不同填充率：100%、90%、75%、50%
5. 对每个填充率，依次从 (0,0,0) 开始，一行行、一层层放置

**优点**：
- 非常快
- 对可以完美填充的零件/材料组合（如 J03 填 L01），直接给出最优解

**缺点**：
- 只能放单一种类零件
- 不考虑利润

**实际贡献**：
- L01 + J03：300×200×150 放 60×50×30 → 最多 5×4×5 = 100个 → **100%利用**
- L03 + J02：200×150×80 放 50×40×40 → 最多 4×3×2 = 24个 → **100%利用**
- L02 + J03：250×150×100 放 60×50×30 → 最多 4×3×3 = 36个 → 利用率 86.4%

### 5.2 方法B：贪心方案（Greedy Patterns）

**思路**：按某种优先级顺序，逐个把零件放到"第一个能放的位置"。

**具体步骤**：
1. 把7种零件按某种标准排序（利润密度、体积、利润等）
2. 从空材料开始，维护已放置零件列表
3. 遍历排序后的零件列表：
   - 在材料中扫描 (x,y,z) 位置
   - 找到第一个不与已有零件重叠、不超边界的位置
   - 放进去，继续下一个
4. 直到没有任何零件能放进去

**多种排序策略**（生成多种不同方案）：
- 按利润密度降序：优先放"单位体积最赚钱的"
- 按体积降序：大件优先
- 按利润降序：单价最高的优先

**优点**：简单直接，能生成混合零件方案

**缺点**：容易卡在局部最优（先放的可能挡住后面更好的）

### 5.3 方法C：极端点方案（Extreme Point Patterns）⭐亮点

这是论文的技术创新点，也是最复杂的算法。核心概念来自 3D Bin Packing 领域的 Extreme Point 概念。

#### 5.3.1 什么是"极端点"？

当你已经放了一些零件后，材料的空余空间不是规则的。**极端点**就是在已放置零件的"角"上产生的候选放置位置。

```
直观理解（2D类比）：
┌──────────────┐
│              │
│    ┌───┐     │
│    │ A │     │  ← 放了一个零件A
│    └───┼──┐  │
│        │B │  │  ← 放零件B时，左下角就是"极端点"
│        └──┘  │
└──────────────┘
```

3D情况下同理：每放一个零件，会产生新的"候选角点"，下一个零件可以放在这些角点上。

#### 5.3.2 算法流程

```
1. 初始化候选点集合 = {(0, 0, 0)}        ← 只有原点
2. 初始化已放置列表 = []
3. 循环直到材料放不下任何零件：
   a. 对每个候选点，尝试放置每种零件×每种姿态
   b. 检查合法性（在材料内、不重叠）
   c. 对合法放置打分：
      分数 = α × 体积填充率 + β × 利润密度
            + γ × 接触面积分 - η × 碎片化惩罚
   d. 束搜索：只保留分数最高的 beam_width 个候选
   e. 选最好的放进去
   f. 更新候选点集合：移除被占用的，添加新的极端点
4. 返回完整的放置方案
```

#### 5.3.3 打分函数

```
总分 = alpha × 体积填充分 + beta × 利润密度分
     + gamma × 接触面积分 - eta × 碎片化惩罚

其中：
- 接触面积分：新零件与已有零件/材料边界的接触面积越大越好
              因为贴边放能减少空间碎片化
- 碎片化惩罚：极端点数量越多→空间越碎→惩罚越大
```

#### 5.3.4 关键参数（在 configs/default.yaml 中）

```yaml
extreme_point:
  alpha: 1.0          # 体积权重
  beta: 0.5           # 利润密度权重
  gamma: 0.3          # 接触面积权重
  eta: 0.1            # 碎片化惩罚权重
  beam_width: 5       # 束搜索宽度（每步保留的候选数）
  random_restarts: 10 # 随机重启次数（每次随机打乱零件顺序）
```

#### 5.3.5 束搜索和随机重启

**束搜索（Beam Search）**：每一步不只保留最好的，而是保留"最好的5个"。这样有更多探索空间，不容易陷入局部最优。

**随机重启**：整个算法跑10次，每次随机打乱零件处理顺序。取10次中最好的方案。这样大大增加了找到好方案的概率。

### 5.4 三种方法的关系和互补

```
网格方案 ──→ 保证纯一种零件的"最佳排布"
              ↓
贪心方案 ──→ 补充"混合零件"的可行方案
              ↓
极端点方案 → 补充"高质量混合方案"（技术亮点）
              ↓
        合并去重 → 每种材料 ~200个方案
```

## 6. 第2步详解：怎么选出最优方案组合

这一步由 `master_solver.py` 实现。

### 6.1 问题转化为整数规划

我们已经有了几百种方案，现在只需要回答：**每种方案用几次？**

设决策变量 `x[t,p]` = 材料类型t使用方案p的次数

### 6.2 问题1的CP-SAT模型

```python
# 目标：最小化浪费
minimize sum( x[t,p] × (材料t体积 - 方案p占用体积) for all t,p )

# 约束：每种材料正好用5块
for each material t:
    sum( x[t,p] for all patterns p of t ) == 5

# 变量范围
x[t,p] >= 0, 整数
```

### 6.3 问题2的CP-SAT模型

```python
# 目标：最大化利润
maximize sum( x[t,p] × 方案p总利润 for all t,p )

# 约束1：每种材料正好用5块
for each material t:
    sum( x[t,p] for all patterns p of t ) == 5

# 约束2：每种零件至少10个
for each piece j:
    sum( x[t,p] × 方案p中零件j的数量 for all t,p ) >= 10

# 变量范围
x[t,p] >= 0, 整数
```

### 6.4 CP-SAT是什么？

CP-SAT = Constraint Programming + SAT (Boolean Satisfiability)

它是 Google OR-Tools 提供的一个求解器，专门用来求解带约束的整数优化问题。它的核心能力：
- 处理整数变量
- 处理线性约束
- 使用SAT技术做搜索和推理
- 能在合理时间内找到精确最优解（对于小到中型问题）

**关键理解**：CP-SAT返回的`OPTIMAL`意味着"它证明了这个解在我们给的变量和约束范围内是最优的"。由于我们的变量只有"方案使用次数"（而不是"每个零件的坐标"），它可以在秒级时间内求解。

### 6.5 求解流程

```python
# master_solver.py 的核心代码
def solve_master_problem1(materials, pieces, all_patterns, config):
    model = cp_model.CpModel()

    # 1. 创建变量
    var_map = {}
    for material_name, patterns in all_patterns.items():
        for p in patterns:
            var_map[(material_name, p.pattern_id)] = model.NewIntVar(0, 5, f"x_{material_name}_{p.pattern_id}")

    # 2. 材料数量约束
    for material_name, material in materials.items():
        model.Add(sum(var_map[(material_name, p.pattern_id)] for p in patterns) == material.count)

    # 3. 目标函数
    total_waste = sum(var_map[key] × waste_of_pattern for key, var in var_map.items())
    model.Minimize(total_waste)

    # 4. 求解
    solver = cp_model.CpSolver()
    solver.parameters.max_time_in_seconds = 120
    status = solver.Solve(model)

    # 5. 提取结果
    return MasterSolution(...)
```

## 7. 结果是怎么算出来的

### 7.1 问题1的结果解读

最终解用了3种方案（每种材料各1种）：

```
L01 → 方案13（J03 × 100个）→ 利用率 100%
L02 → 方案97（J02 × 30个 + J03 × 20个）→ 利用率 96%
L03 → 方案167（J02 × 24个）→ 利用率 100%
```

**总利用率 = (45000000+18000000+12000000) / 75750000 = 99.01%**

**为什么L01和L03能100%？**
- L01 = 300×200×150，J03 = 60×50×30
  - 长方向：300/60 = 5（整除）
  - 宽方向：200/50 = 4（整除）
  - 高方向：150/30 = 5（整除）
  - 完美填满！
- L03 = 200×150×80，J02 = 50×40×40
  - 长方向：200/50 = 4
  - 宽方向：150/40 = 3.75... ✗
  - 但其实 J02 可以旋转！40×50×40 宽方向 150 除不尽
  - 具体怎么放的要查方案167的坐标数据

**为什么L02只有96%？**
- L02 = 250×150×100
- 最优方案：J02×30 + J03×20 = 3,600,000 / 3,750,000 = 96%
- L02搜索了736种方案，都无法超过96%
- **经验上很可能是全局最优**，但没有数学证明

### 7.2 问题2的结果解读

用了7种方案（1个L01 + 4个L03 + 2个L03...具体看solution.json）：

```
总利润 = 748,640
利用率 = 96.15%（比问题1低，因为要满足每种≥10的约束）

各零件产量：
J06: 370个（利润密度高 + 容易放 → 大量生产）
J07: 354个（利润密度最高）
J03: 100个
J01, J02 各30个（满足≥10即可，利润密度低不合算）
J04: 12个, J05: 14个（刚刚超过10）
```

**策略解读**：求解器自动发现应该多切 J06 和 J07（利润密度高且形状细长好放），J01/J02/J04/J05 满足最低要求就不再切了。这是一个智能的权衡。

### 7.3 关键指标计算公式

```python
# 利用率
利用率 = 已用体积 / 总体积

# 问题1 gap（到100%的距离）
gap = 1.0 - 利用率
# 当前: 1.0 - 0.9901 = 0.0099(0.99%)

# 问题2 理论利润上界（松弛的，放得很宽）
利润上界 = 总体积 × 最高利润密度
# = 75,750,000 × 0.01125（J07的利润密度）
# = 852,187.5

# 问题2 gap（到松弛上界的距离）
gap_ratio = (852187.5 - 748640) / 852187.5 = 12.15%
# ⚠️ 这个12.15%不代表我们离最优还有12.15%
# 因为上界本身是"不可能达到的宽松上界"
```

## 8. 最优性证明是怎么回事

### 8.1 三个层次的理解

```
第1层：方案库内最优（CP-SAT OPTIMAL）
  "在生成的200个方案里，这个组合是最好的"
  ✅ 我们已经达到
  方法：CP-SAT返回OPTIMAL就是

第2层：经验最优（实证最优）
  "我们试了所有能想到的方法，都没找到更好的"
  ✅ 我们已经达到
  证据：网格、贪心、极端点 × 多种参数、L02专项搜索736种方案
       全部卡在同样的上限

第3层：数学证明的全局最优
  "我们用数学方法严格证明了没有任何方案能更好"
  ⚠️ 对L01和L03是✅，对L02是❌（尚未完成）
  方法：枚举所有可能的生产向量 → 逐一用CP-SAT检查几何可行性
```

### 8.2 证书模块做了什么

`certificate.py` + `exact_enumeration.py` + `exact_geometry_cp.py` 的合作：

```
1. exact_enumeration.py：
   枚举L02上所有可能的生产向量 (a1,a2,...,a7)
   其中 aj = 零件j的个数
   约束：sum(aj × 体积j) ≤ L02体积(3,750,000)
         sum(aj × 体积j) > 3,600,000（超过当前最优）
         且 aj 都是非负整数

   比如：(0,0,40,0,0,0,0) 表示放40个J03 = 40×90000 = 3,600,000 ← 刚好达到
        (0,0,41,0,0,0,0) = 3,690,000 ← 超过当前！需要检查

2. exact_geometry_cp.py：
   对每个"超过3,600,000"的向量，用CP-SAT检查：
   "是否存在一种几何摆放方式，使得这个向量可行？"
   - 用离散坐标变量
   - 用orientation选择变量
   - 用不重叠、不超界的约束

3. certificate.py：
   汇总结果：
   - 如果所有超过基线的向量都是INFEASIBLE → PROVED_OPTIMAL
   - 如果有FEASIBLE的 → 存在更好的方案 → NOT_PROVED
   - 如果有UNKNOWN（超时）→ 不确定 → NOT_PROVED
```

### 8.3 当前状态

```
问题1:
  L01: 100% → 自动最优 ✅
  L03: 100% → 自动最优 ✅
  L02: 96% → 未证明最优（证书生成未完成/超时） ⚠️
```

## 12. 常见疑问解答

### Q1: 为什么"OPTIMAL"不是真的全局最优？

**A**: 因为求解器只看到我们生成的~200个方案。如果存在一个我们没生成的"神方案"（比如以一种我们没试过的零件组合达到更高利用率），求解器看不到它。

类比：你让小孩从冰箱里的菜中选最好吃的，他说"排骨最好吃"——这是"冰箱内最优"。但如果有他没见过的和牛，那结论就变了。

### Q2: 99.01%利用率好不好？

**A**: 非常好。2/3的材料100%利用，总浪费仅0.99%。在3D切割问题中，能达到这个水平说明方法很有效。

### Q3: 问题2的gap 12.50%是不是很大？

**A**: 不大。这个12.50%是相对一个"不可能达到"的上界。那个上界假设所有体积都用利润密度最高的J07(0.01125)填满，但J07是120×20×20的细长条，几何上不可能完美填满所有材料。真实的最优性gap远小于12.50%。

### Q4: 极端点算法相比贪心好在哪？

**A**: 贪心是"一条道走到黑"——每次选最佳位置放一个零件，放完不管了。极端点是"多条路同时探索"——维护候选点集，每次选最佳组合，加上束搜索和随机重启，有更大机会找到全局更好的方案。

### Q5: 如果我是小白，最应该看懂哪几个文件？

**A**:
1. `models.py` — 理解数据结构
2. `pattern_generator.py` — 理解方案怎么生成的
3. `master_solver.py` — 理解怎么选最优组合
4. `experiment.py` — 理解整个流程怎么串起来

### Q6: 我想改进结果，应该动哪里？

**A**:
- 调参数：`configs/default.yaml` 的 `extreme_point` 部分（beam_width, random_restarts）
- 改算法：`extreme_point.py` 的打分函数
- 加方法：在 `pattern_generator.py` 添加新的方案生成器
- L02突破：跑 `--with-exact-certificate` 完整版确认是否真的不能超过96%

## 附：快速概念速查表

| 概念 | 简单解释 |
|------|----------|
| **Pattern（方案）** | 一块材料的具体切割方式 = 哪些零件放在哪些位置 |
| **CP-SAT** | Google的整数规划求解器 |
| **极端点** | 已放零件角上产生的候选放置位置 |
| **束搜索** | 每步保留N个最佳候选，不是只保留1个 |
| **利润密度** | 利润÷体积，衡量"每单位体积能赚多少钱" |
| **利用率** | 已用体积÷总体积 |
| **Gap** | 当前解与某个上/下界的差距 |
| **消融** | 逐个去掉组件，看哪个组件带来提升 |
| **敏感性** | 改变参数，看结果变化大不大 |
| **全局最优** | 所有可能方案中的最好方案（不限于我们生成的） |
| **方案库最优** | 我们生成的方案中的最好方案 |
